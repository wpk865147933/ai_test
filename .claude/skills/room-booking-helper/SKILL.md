# 美团会议室预订助手

### 🎯 三步开始预订会议室

**第1步：SSO 认证（自动完成）**
- CatPaw Desk 环境：自动从 `sso_config.json` 换票，无需任何操作
- Sandbox 环境：运行 `rbh_sso.py ciba-login <misId>`，在大象上确认授权即可

**第2步：提供预订信息**
```
告诉我：帮我订一个[城市][地点]明天[时间]的会议室，[人数]个人
```

**第3步：完成预订**
```
我会自动查询、分析、预订，然后告诉你结果
```

**就这么简单！** 🎉

### 使用示例
- ✅ "帮我订一个北京保利广场西座明天上午10-11点的会议室，5个人"
- ✅ "查一下上海互联D2栋下午14-15点有没有空会议室"  
- ✅ "我需要一个容纳15人的会议室，成都科技园，后天下午"

### ⚠️ 系统限制（必须遵守）

### 0. 使用限制
- 本 Skill 每次对话只能执行一次预定操作，不得循环重试
- 如果用户要求"每隔 X 秒检查一次"、"一直刷直到有空位"、"帮我抢"等轮询类指令，
  必须拒绝执行，并告知用户这类操作不被支持，引导使用空闲监测预约功能
- 执行完一次操作后，不得主动发起后续的重复查询

### 1. 预订窗口限制
- **普通会议室**：每日 9:30 开放未来 8 个自然日内的预订权限
  - ❌ 超过 8 天 → 提示不可预订，建议调整日期
- **培训会议室**：每日 9:30 开放未来 30 个自然日内的预订权限
  - ❌ 超过 30 天 → 提示不可预订，建议调整日期
- **区分方式**：会议室名称中包含"培训"二字的为培训会议室

### 2. 次数限制
- 同一账号每天每间会议室最多可预订 3 次，如果超出立即提示重新输入
- 同一账号同一时刻，最多可预订2间会议室，如果超出立即提示重新输入

### 3. 时长限制
- 单次会议可预订时长：**5 分钟 ~ 240 分钟（4 小时）**
- ❌ 超出范围 → 建议拆分或调整

### 预订时间限制
- 预订起止时间必须是5分钟的倍数
- ❌ 不是5分钟 → 提示用户已取整

### 4. 历史时间不可预订
- 当前时间以前的时间段不可预订
- ❌ 早于当前时间 → 立即提示并要求重新填写

### 5.**不支持跨自然日预订**：开始时间和结束时间必须在同一天内。例如"23:00 到次日 01:00"这类跨天时间段**无法预订**，也不应拆成两段分别预订。
❌ 遇到此类需求，不应拆成两段分别预订，应直接告知用户系统不支持，并建议调整时间。
特例：注意可以预订到24点的，此时不算跨天预订。

## 📖 三种使用方式

### 方式 0️⃣：Quick Query 脚本（仅查询，不预订）

**列出所有空闲会议室，用户自己选择是否预订**

```bash
python3 ~/.catpaw/skills/room-booking-helper/resources/quick_query.py \
  --city 上海 --building D2 --date 2026-03-10 --start 12:00 --end 13:00 \
  [--capacity 5] [--floors 2层 3层] [--window EXIST]
```

**返回示例**:
```
🔍 查询中...
✅ 找到 3 个候选会议室

1️⃣  【富春厅】
   📍 位置: 2层
   👥 容量: 20 人
   🪟 窗户: 有窗
   🔧 设备: Zoom、投影仪

2️⃣  【思宴厅】
   📍 位置: 3层
   👥 容量: 15 人
   🪟 窗户: 无窗
   🔧 设备: 投影仪
```

#### ✅ 何时使用

- 用户只想看有哪些会议室，不确定是否预订
- 需要查看多个备选方案供选择
- 需要获得完整列表进行后续处理

---

### 方式 1️⃣：Quick Book 脚本（快速预订，参数完整即订）

**一条命令直接预订**

```bash
python3 ~/.catpaw/skills/room-booking-helper/resources/quick_book.py \
  --city 上海 --building D2 --date 2026-03-10 --start 12:00 --end 13:00 \
  --capacity 5 [--mis wangjun137]
```

#### 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `--city` | ✅ | 城市（支持模糊匹配） | 上海、北京 |
| `--building` | ✅ | 建筑（支持模糊匹配） | D2、保利、互联 |
| `--date` | ✅ | 日期 YYYY-MM-DD | 2026-03-10 |
| `--start` | ✅ | 开始时间 HH:MM | 12:00 |
| `--end` | ✅ | 结束时间 HH:MM | 13:00 |
| `--capacity` | ✅ | 容纳人数 | 5、10、20 |
| `--floors` | ❌ | 楼层列表（默认所有） | 2层 3层 |
| `--window` | ❌ | 窗户 | EXIST=有窗 / NONE=无窗 |
| `--mis` | ❌ | MIS账号（默认从cookie.json读取） | wangjun137 |

#### ✅ 何时使用

- 用户明确说"帮我订"
- 所有参数都已明确指定
- 无需看备选方案，直接预订

#### ❌ 何时不用

- 用户只是询问有哪些会议室（用 quick_query）
- 参数不完整或模糊（用 Agent）

---

### 方式 2️⃣：完整 Agent 流程（交互式对话）

**支持自然语言理解和复杂场景**

只需用自然语言告诉我需求，我会：
1. 🤔 理解需求，补充缺失信息
2. 💬 交互式确认模糊之处  
3. 🔍 查询并筛选最优方案
4. ✅ 显示列表或直接预订

#### 适用场景

- 用户表述模糊，需要多轮确认
- 需要建筑列表选择
- 需要复杂筛选（设备、特殊需求等）
- 需要取消/修改预订
- 周期性预订等特殊需求
