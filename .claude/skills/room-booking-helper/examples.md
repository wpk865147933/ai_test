## 示例 5.5：跨午夜预订（23:00-00:00）

**用户请求**：
```
帮我订个北京保利西座明天23:00-00:00的会议室，5个人
```

**助手处理**：

### 自动处理跨午夜时间
```
💬 检测到您的预订跨越午夜（23:00-00:00）

✅ 系统会自动处理：
- 开始时间：明天 23:00
- 结束时间：后天 00:00（自动转换为第二天的零点）
- 时长：60 分钟 ✅（符合 5-240 分钟的要求）

继续查询空闲会议室...
```

查询结果：
```
🔍 2026-02-19 23:00-2026-02-20 00:00 可用会议室（北京/保利广场西座）：

📍 鹤河厅（2层）
   - 容量：6 人 ✅
   - 设备：Zoom
   - 状态：23:00-00:00 空闲

🎉 预订成功！
- 会议室：鹤河厅
- 日期：2026-02-19 23:00 ~ 2026-02-20 00:00
- 容量：6 人
- 位置：北京/保利广场西座/2 层
```

**关键特性**：
- ✅ 允许 23:00-24:00 的夜间跨午夜预订
- ✅ 若 end_time < start_time（如 23:00-01:00），系统自动调整为第二天
- ✅ 若 end_time=00:00 且 start_time>00:00，自动转换为第二天的 00:00
- ⏰ 仍然遵守 5-240 分钟的时长限制

## 示例 10：真实 API 调用流程

**用户请求**：
```
帮我在北京这周六上午10点订一个会议室
```

**完整的 API 调用过程**：

### 1️⃣ 第一步：获取城市和大厦列表
```bash
curl -X POST https://calendar.sankuai.com/room/front/app/room/cityBuilding \
  -H 'Content-Type: application/json' \
  -H 'Cookie: [浏览器cookie]' \
  -d '{}'
```

**响应**：返回北京的所有大厦，如"北京/保利广场西座"（ID: 843）

### 2️⃣ 第二步：查询会议室列表
```bash
curl -X POST https://calendar.sankuai.com/meeting/api/pc/room/appointment/v2/find-rooms \
  -H 'Content-Type: application/json' \
  -H 'Cookie: [浏览器cookie]' \
  -d '{
    "date": 1772848800000,
    "buildingId": "843",
    "floorIds": [],
    "capacity": [{"capacityMin": 7, "capacityMax": 14, "name": "7-14人"}],
    "startTime": "10:00",
    "endTime": "11:00"
  }'
```

**响应**：返回可用会议室列表（敦煌厅、富春厅等）

### 3️⃣ 第三步：查询已有预订
```bash
curl -X POST https://calendar.sankuai.com/meeting/api/pc/room/appointment/findRoomAppointmentsV2 \
  -H 'Content-Type: application/json' \
  -H 'Cookie: [浏览器cookie]' \
  -d '{
    "date": 1772848800000,
    "roomIds": [11485, 11486, 11490]
  }'
```

**响应**：检查这些会议室在 10:00-11:00 是否有冲突

### 4️⃣ 第四步：提交预订
```bash
curl -X POST https://calendar.sankuai.com/api/v2/xm/schedules \
  -H 'Content-Type: application/json' \
  -H 'Cookie: [浏览器cookie]' \
  -d '{
    "title": "工作会议",
    "startTime": 1772848800000,
    "endTime": 1772852400000,
    "isAllDay": 0,
    "location": "",
    "attendees": ["6485487"],
    "noticeType": 0,
    "noticeRule": "P0Y0M0DT0H10M0S",
    "recurrencePattern": {"type": "NONE", "showType": "NONE"},
    "deadline": 0,
    "memo": "",
    "organizer": "6485487",
    "room": {
      "id": 11486,
      "name": "富春厅",
      "email": "room-blx-2f-a-fuchunting@meituan.com",
      "capacity": 13,
      "floorId": 2001,
```

## 示例 11：常见错误及处理

### ❌ 错误 1：时间参数不合法
```json
{
  "data": {"message": "时间参数不合法"},
  "rescode": 1
}
```
**原因**：时间戳格式不正确（需要毫秒级）或时区计算错误
**处理**：使用正确的时间戳计算方法

### ❌ 错误 2：过去时间不可预订会议室
```json
{
  "data": {"message": "过去时间不可预订会议室"},
  "rescode": 1
}
```
**原因**：用户选择的时间已经过去
**处理**：提示用户选择未来的时间

### ❌ 错误 3：会议室已被预订
```json
{
  "data": {"message": "会议室已被预订，请重新选择！"},
  "rescode": 1
}
```
**原因**：所选会议室在该时段已被预订
**处理**：从查询结果中选择其他空闲会议室

### ❌ 错误 4：仅支持历史7天内的数据查询
```json
{
  "data": {"errorCode": 1559, "message": "仅支持历史7天内的数据查询"},
  "status": 0
}
```
**原因**：查询日期超过 7 天
**处理**：提示用户只能预订 7 天内的会议室
