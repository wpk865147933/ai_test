# 学城知识库搜索技能

## 搜索模式判断（首要步骤）

收到用户请求后，**首先判断搜索模式**，后续流程依模式分支执行：

| 用户意图 | 触发关键词示例 | 执行模式 |
|----------|--------------|----------|
| 搜索学城内部文档 | "学城搜索"、"KM上找"、"内部文档" | **模式 A：学城搜索** |
| 联网搜索公网信息 | "联网搜索"、"Google一下"、"百度"、"搜索互联网"、"网上找" | **模式 B：联网搜索** |
| 两者都要 | "学城和网上都搜一下" | **模式 A + B 合并** |

---

## 模式 A：学城搜索

### A-1. 理解搜索意图

分析用户的问题，提取核心搜索关键词。策略：

- 从用户问题中提取 1-3 个核心关键词
- 如果用户问题较宽泛，拆分为多个搜索维度分别搜索
- 优先使用中文关键词（学城文档以中文为主），技术术语可保留英文

### A-2. 执行搜索

使用 `km_search` 工具搜索学城知识库：

```
km_search(search_term="核心关键词")
```

搜索策略：

- **单一主题**：直接搜索核心关键词，如 `数据倾斜 解决办法`
- **多维度主题**：分多次搜索，每次聚焦一个维度。例如用户问"Spark性能优化"，可分别搜索 `Spark 内存调优`、`Spark shuffle优化`、`Spark 数据倾斜`
- **结果不足时**：换用同义词或相关词重新搜索，如 `数据治理` → `数据质量管理`

### A-3. 内容整理与总结

对搜索结果按以下结构整理输出：

```
## 搜索结果总结

[用 2-3 句话概括搜索到的核心结论]

### 要点一：[主题名]

[整理该主题下的关键内容，合并多篇文档中的相关信息，去重并提炼]

### 要点二：[主题名]

[同上]

...

### 参考文档

- [文档标题1](学城链接1) — 一句话说明文档价值
- [文档标题2](学城链接2) — 一句话说明文档价值
```

**整理原则**：

- 合并多篇文档中的重复内容，提炼核心观点
- 保留关键的代码片段、参数配置、命令等实操内容
- 按主题/场景组织，而非按文档来源组织
- 标注信息来源，便于用户追溯原文
- 每篇参考文档附上学城链接（格式：`https://km.sankuai.com/collabpage/xxx`）

### A-4. 补充搜索（按需）

如果用户对某个要点想深入了解，可以：

- 针对该要点进行更精确的二次搜索
- 使用 `web_fetch` 获取特定学城文档的完整内容（如果链接可访问）

### A-5. 生成学城文档（按需）

整理完成后，**询问用户是否需要将内容保存为一篇新的学城文档**。若用户确认，参考"第 6 步：创建学城文档"执行。

## 模式 B：联网搜索

> 触发条件：用户明确说"联网搜索"、"Google"、"百度"、"网上找"等关键词。

### B-1. 理解搜索意图

同模式 A-1，提取核心关键词，必要时拆分为多个搜索维度。

### B-2. 执行联网搜索

使用 `web_search` 工具进行公网搜索：

```
web_search(query="核心关键词")
```

搜索策略：

- 优先使用中英文双语关键词，技术类问题英文效果更好
- 多维度主题分多次搜索，每次聚焦一个维度
- 结果不足时换用同义词或相关词重试

获取到搜索结果后，对**摘要内容丰富、原始链接可访问**的条目，使用 `web_fetch` 进一步抓取页面正文：

```
web_fetch(urls=["https://原始链接"])
```

- 每次最多抓取 3 个链接，优先选择内容最相关的
- 若 `web_fetch` 失败，仅使用搜索摘要，并在输出中注明"未能获取全文"

### B-3. 内容整理与总结

对联网搜索结果按以下结构整理输出，**必须保留每条信息的原始来源链接**：

```
## 联网搜索结果总结

[用 2-3 句话概括搜索到的核心结论]

### 要点一：[主题名]

[整理该主题下的关键内容，合并多个来源中的相关信息，去重并提炼]

来源：[网页标题](原始URL)

### 要点二：[主题名]

[同上]

来源：[网页标题](原始URL)

...

### 参考链接

- [网页标题1](原始URL1) — 一句话说明内容价值
- [网页标题2](原始URL2) — 一句话说明内容价值
- [网页标题3](原始URL3) — 一句话说明内容价值
```

**整理原则**：

- **必须保留原始 URL**，每个要点下标注来源，参考链接列表完整列出所有引用页面
- 合并多个来源中的重复内容，提炼核心观点
- 保留关键的代码片段、命令、配置等实操内容
- 注明信息时效性（如"截至 {搜索日期}"），提醒用户核实最新状态

### B-4. 自动生成学城文档

**联网搜索模式下，整理完成后直接进入学城文档创建流程，无需额外询问用户。**

文档内容在 B-3 整理结果基础上，补充以下元信息头部：

```markdown
> 本文由 CatPaw 通过联网搜索自动整理生成，生成时间：{YYYY-MM-DD}。
> 内容来源于公开网络，信息时效性请以原始链接为准。
> 所有参考链接见文末"参考链接"列表。

---

[正文内容，与 B-3 输出保持一致，原始链接完整保留]
```

然后参考"第 6 步：创建学城文档"执行文档创建。

## 第 6 步：创建学城文档（通用）

无论来自模式 A 还是模式 B，创建学城文档的流程如下：

### 6.1 确认必要信息

在创建文档前，需向用户确认（若用户已提供则跳过）：

- **文档标题**：默认使用搜索主题作为标题，如"[主题] 知识整理"或"[主题] 联网搜索整理"
- **存放位置**：父文档 ID（`--parent PARENT_ID`）或空间 ID（`--space SPACE_ID`）
  - 若用户不确定，可先运行 `km hierarchy-info` 查看个人空间结构，辅助用户选择

### 6.2 环境准备

```bash
source ~/.meituan-local-tools/.venv/bin/activate
```

### 6.3 创建文档

```bash
# 将内容写入临时文件
cat > /tmp/km_search_result.md << 'KMEOF'
{文档完整 Markdown 内容，原始链接完整保留}
KMEOF

# 创建学城文档
km create --title "{文档标题}" --file /tmp/km_search_result.md --parent {PARENT_ID}

# 清理临时文件
rm -f /tmp/km_search_result.md
```

### 6.4 反馈结果

文档创建成功后，向用户展示：

```
✅ 学城文档已创建！

📄 文档标题：{标题}
🔗 文档链接：https://km.sankuai.com/collabpage/{新文档ID}
📁 存放位置：{父文档标题 或 空间名称}
```

若创建失败（如权限不足、父文档 ID 无效），提示用户检查：
- 是否已在浏览器登录 https://km.sankuai.com（SSO 认证）
- 父文档 ID 是否正确，且当前账号有"添加"权限
- 运行 `source ~/.meituan-local-tools/.venv/bin/activate` 确保环境已激活

---

## 注意事项

- 搜索结果可能被截断，注意提示用户可查看原文获取完整内容
- 学城文档的时效性不同，优先参考近期更新的文档
- 如果搜索无结果，建议用户调整关键词或直接访问学城搜索
- 始终提供原文链接，方便用户进一步阅读
- 生成学城文档前，务必确认父文档 ID 或空间 ID，避免文档创建到错误位置
- 临时文件（`/tmp/km_search_result.md`）在文档创建成功后必须删除
- **联网搜索模式下**，原始 URL 必须完整保留在学城文档中，不得省略或替换
