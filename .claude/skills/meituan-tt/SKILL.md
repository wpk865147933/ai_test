# 美团 TT 工单分析 Skill

## When to use
- 用户要求分析 TT 工单
- 用户说"帮我看看这个工单"
- 用户发送 TT 工单链接
- 用户要求统计工单数据

## When not to use
- 用户只是要创建工单（去 TT 系统）
- 用户要修改工单状态（去 TT 系统）

## Required inputs
- 工单链接或工单编号
- （可选）分析维度：按类型、按时间、按处理人等

## Procedure

### Phase 1: 获取工单信息
```bash
# 如果有 tt CLI 工具
tt get <ticket_id>

# 如果没有 CLI，通过浏览器访问
# https://tt.sankuai.com/ticket/detail?tid=<ticket_id>
```

### Phase 2: 分析工单内容
提取关键信息：
1. **工单类型**：Bug、需求、咨询、投诉等
2. **工单状态**：待处理、处理中、已解决、已关闭
3. **优先级**：P0、P1、P2、P3
4. **处理时长**：创建时间、首次响应时间、解决时间
5. **根因分类**：系统问题、用户操作、外部依赖、代码 Bug 等

### Phase 3: 统计分析（多工单场景）
1. **按类型统计**：哪种类型的工单最多？
2. **按时间段统计**：高峰期是什么时候？
3. **按处理人统计**：谁处理的工单最多？
4. **按优先级统计**：P0/P1 工单占比多少？

### Phase 4: 输出分析报告
```markdown
## TT 工单分析报告

**分析时间范围**: YYYY-MM-DD ~ YYYY-MM-DD
**工单总数**: xx 个

### 📊 工单概览
- P0 工单：xx 个（xx%）
- P1 工单：xx 个（xx%）
- P2 工单：xx 个（xx%）
- P3 工单：xx 个（xx%）

### 📈 趋势分析
- 日均工单量：xx 个
- 高峰期：周x / xx:00-xx:00
- 环比变化：↑xx% / ↓xx%

### 🏆 处理效率
- 平均首次响应时间：xx 分钟
- 平均解决时长：xx 小时
- 超时工单占比：xx%

### 🔍 根因分析
| 根因类型 | 工单数 | 占比 | 典型案例 |
|---------|--------|------|---------|
| 系统 Bug | xx | xx% | xxx |
| 用户操作 | xx | xx% | xxx |
| 外部依赖 | xx | xx% | xxx |
| 代码问题 | xx | xx% | xxx |

### 💡 改进建议
1. **高频问题**：xxx → 建议 xxx
2. **处理效率**：xxx → 建议 xxx
3. **预防措施**：xxx → 建议 xxx

### 📋 待关注的工单
1. [工单号] 标题 - 状态 - 已超时 xx 小时
2. ...
```

## Output format
- Markdown 格式的分析报告
- 包含数据统计、趋势分析、根因分类、改进建议
- 可选：输出 Excel/CSV 格式的详细数据

## Failure handling
1. 如果工单不存在或无权限访问 → 告知用户原因
2. 如果无法获取工单信息 → 检查登录状态
3. 如果数据量太大 → 建议缩小时间范围或分批分析

## Notes
- 当前没有 tt CLI 工具，需要通过浏览器访问或申请 API 权限
- 建议后续接入 TT API 或开发 CLI 工具
